## Detailed Problem

Standard CNN-based lane detectors produce dense pixel-level predictions that degrade significantly under occlusion — shadows from overpasses, tight curves where only partial lane markings are visible, and adverse lighting conditions. The failure mode is systematic: the detector over-predicts straight-line geometry in conditions where curvature requires extrapolation beyond visible evidence.

The goal was to build a refinement layer that operates on an existing detector's output, correcting geometric errors without retraining the upstream model.

## Architecture

The pipeline is two-stage:

1. **Perception stage** — A MobileNetV3 backbone extracts lane-point predictions from the input image frame.
2. **Refinement stage** — A Double DQN agent takes the raw prediction and a cropped image patch as state, and learns to apply incremental corrections to individual lane points.

The agent's action space is discrete: shift each point left, right, up, down, or leave unchanged. The reward signal is the negative lateral error against ground-truth annotations from the dataset.

## Algorithm & Code Design

```python
def compute_reward(predicted_points, ground_truth_points):
    lateral_error = torch.mean(
        torch.abs(predicted_points[:, 0] - ground_truth_points[:, 0])
    )
    return -lateral_error.item()
```

<Callout type="insight" title="Reward shaping">
The reward only penalizes lateral error. Adding a curvature smoothness penalty in later experiments reduced oscillation artifacts on tight radii — a follow-up worth formalizing.
</Callout>

## Results

The refined pipeline improved lane-point accuracy by 18% on the occluded test split relative to the standalone CNN baseline. Curvature error on tight-radius segments dropped 23%. Inference overhead from the refinement stage was 8ms per frame, keeping the combined latency within the 40ms real-time budget.

## Next Steps

The reward function could be extended to penalize geometric implausibility (non-monotonic curvature) in addition to lateral error. Applying the same two-stage pattern to a transformer-based backbone is a natural next experiment.
